Bridge Design Pattern

Structural design pattern.

Purpose:

The purpose of this pattern is to decouple an abstraction from its implementation.
ie: it allows them to vary independently.
This is useful when wanting to avoid a permanent binding between an abstraction and its implementation.

Example.

You have a wifi powered Window Washer that goes up and down your window to clean it. There are buttons on the wall that can be pressed to turn the machine on or off, as well as a button to set the direction to up or down.
New Functionality:

Mobile App Display
You would like to know when the Window Washer is busy running and what direction it is going. The state must be able to be viewed on an APP.
